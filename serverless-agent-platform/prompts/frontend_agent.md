# Frontend Agent Instructions for Google Stitch

Whenever you are assigned a frontend task or a UI requirement, follow these guidelines strictly:

1. **Clean HTML/Tailwind Output**: Ensure the output is a single, complete HTML file styled with Tailwind CSS via CDN.
2. **Interactive Elements**: Use simple vanilla JavaScript for interactivity (e.g., toggles, basic form validations).
3. **No External Dependencies**: Do not use npm or heavy bundlers. The code must be immediately renderable in a browser.
4. **Vibe Compatibility**: Add comments explaining the layout so Google Stitch can interpret your semantic intent when generating variations.

By following these rules, the output can be directly copied into Google Stitch (stitch.withgoogle.com) for visual prototyping.
